# JVS (Java Version Switch) 使用文档

轻量级 macOS / Linux Java 版本管理工具，用 Rust 编写。

---

## 1. 安装

```bash
git clone <仓库地址>
cd java-version-switch
./install.sh
```

安装脚本会自动完成以下操作：
- 检测并安装 Rust 工具链（如未安装）
- 编译项目
- 将 `jvs` 可执行文件复制到 `/usr/local/bin/`
- 在 shell 配置文件（`~/.zshrc` 或 `~/.bashrc`）中添加自动加载配置和 shell 函数包装器

安装完成后，重新加载 shell：

```bash
source ~/.zshrc
```

---

## 2. 卸载

```bash
./uninstall.sh
```

卸载脚本会自动完成以下操作：
- 删除 `/usr/local/bin/jvs` 可执行文件
- 删除 `~/.jvs/` 配置目录
- 从 shell 配置文件（`~/.zshrc` 或 `~/.bashrc`）中移除 JVS 配置块

## 3. 命令说明

### 3.1 查看已安装的 Java 版本

```bash
jvs list
```

输出示例：

```
Installed Java versions:
   17.0.4.1     (Oracle Corporation)  /Library/Java/JavaVirtualMachines/jdk-17.0.4.1.jdk/Contents/Home
   17.0.14      (Microsoft)           ~/Library/Java/JavaVirtualMachines/ms-17.0.14/Contents/Home
 * 11.0.20      (Oracle Corporation)  /Library/Java/JavaVirtualMachines/jdk-11.jdk/Contents/Home
   1.8.0_341    (Unknown)             /Library/Java/JavaVirtualMachines/jdk1.8.0_341.jdk/Contents/Home
```

`*` 标记表示当前正在使用的版本。

### 3.2 查看当前 Java 版本

```bash
jvs current
```

输出示例：

```
Current Java: 11.0.20 (Oracle Corporation)
  JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-11.jdk/Contents/Home
```

### 3.3 切换 Java 版本

```bash
jvs use <版本前缀>
```

支持前缀匹配，例如：

| 命令 | 匹配版本 |
|------|----------|
| `jvs use 17` | 匹配 17.x.x（如有多个 17 版本会提示选择更精确的前缀） |
| `jvs use 17.0.4` | 精确匹配 17.0.4.1 |
| `jvs use 11` | 匹配 11.0.20 |
| `jvs use 1.8` | 匹配 1.8.0_341 |

通过 `install.sh` 安装后，切换会立即生效，无需手动 source。

> **注意**：如果直接运行二进制文件（未通过 shell 函数），需要手动执行 `source ~/.jvs/env` 使切换生效。

### 3.4 查看帮助和版本

```bash
jvs --help     # 查看帮助信息
jvs --version  # 查看工具版本
```

### 3.5 安装 JDK

```bash
jvs install <主版本号>
```

通过系统包管理器安装 JDK：

| 平台 | 包管理器 | JDK 来源 |
|------|---------|----------|
| macOS | brew | Eclipse Temurin (`temurin@{version}`) |
| Linux | apt | OpenJDK (`openjdk-{version}-jdk`) |
| Linux | yum | OpenJDK (`java-{version}-openjdk-devel`) |

示例：
```bash
jvs install 21    # 安装 JDK 21
jvs list          # 查看安装结果
jvs use 21        # 切换到新版本
```

### 3.6 卸载 JDK

```bash
jvs remove <主版本号>
```

通过系统包管理器卸载 JDK。

示例：
```bash
jvs remove 21
```

## 4. JDK 扫描路径

工具会根据操作系统自动扫描以下目录中的 JDK 安装：

**macOS：**

| 路径 | 说明 |
|------|------|
| `/Library/Java/JavaVirtualMachines/` | 系统级安装（Oracle、Adoptium 等安装包默认位置） |
| `~/Library/Java/JavaVirtualMachines/` | 用户级安装（如 Microsoft OpenJDK） |

**Linux：**

| 路径 | 说明 |
|------|------|
| `/usr/lib/jvm/` | 系统包管理器安装（apt、yum 等） |
| `/usr/java/` | Oracle RPM 安装默认位置 |
| `/opt/java/` | 手动安装常用位置 |
| `~/.sdkman/candidates/java/` | SDKMAN 安装 |
| `~/.jdks/` | IntelliJ IDEA 下载的 JDK |

只要 JDK 目录下存在 `release` 文件，即可被识别。

## 5. 配置文件

| 文件 | 用途 |
|------|------|
| `~/.jvs/config.json` | 保存当前选择的 Java 版本 |
| `~/.jvs/env` | shell 环境变量文件，设置 JAVA_HOME 和 PATH |

`config.json` 示例：

```json
{
  "current": "/Library/Java/JavaVirtualMachines/jdk-11.jdk/Contents/Home"
}
```

---

## 6. 工作原理

1. 扫描两个标准 JDK 安装目录，读取每个 JDK 的 `release` 文件获取版本和厂商信息
2. 切换时将选中的 JAVA_HOME 写入 `~/.jvs/env`，同时清除 PATH 中残留的旧 Java 路径
3. 将选择持久化到 `~/.jvs/config.json`
4. shell 配置文件中的 `jvs()` 函数包装器在执行 `jvs use` 后自动 source 环境变量文件，实现即时切换
5. shell 启动时自动 `source ~/.jvs/env`，加载上次选择的版本

## 7. 常见问题

**Q: 切换后 `java -version` 没变？**
A: 如果通过 `install.sh` 安装，切换应该是自动的。如果仍未生效，可能是直接运行了二进制文件而非 shell 函数，请手动执行 `source ~/.jvs/env`，或打开新终端窗口。

**Q: `jvs use 17` 报错说匹配到多个版本？**
A: 系统中安装了多个 17.x 版本，请使用更精确的前缀，如 `jvs use 17.0.4` 或 `jvs use 17.0.14`。

**Q: 新安装的 JDK 没有被识别？**
A: 确认 JDK 安装在上述扫描路径之一，且目录下存在 `release` 文件。

**Q: 需要 sudo 权限吗？**
A: 仅安装时复制二进制文件到 `/usr/local/bin/` 需要 sudo，日常使用不需要。

**Q: `jvs install` 支持哪些包管理器？**
A: macOS 支持 Homebrew，Linux 支持 apt-get 和 yum。安装前请确保对应的包管理器已安装。
